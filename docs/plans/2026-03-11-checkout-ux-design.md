# Checkout 页面优化设计（独立地址页）

## 目标
- 在不打断“生产单导出”主流程的前提下，加入电商式选购与结算体验。
- 增强尺码选择、数量配置、物流与地址填写能力。
- 保持视觉风格一致（研报风/干净、强调信息层级）。

## 页面结构与流转
- 采用同组件内的双视图切换：
  - `结算页`（默认）
  - `地址页`（独立页面样式，保存后返回）
- 物流模式切换：
  - 点击“物流配送”进入地址页填写地址
  - “现场自提”隐藏地址卡片，仅保留备注

## 结算页模块
- 商品预览：正面/背面切换 + 缩略图高亮
- 价格展示：单价、合计（数量变动实时更新）
- 推荐尺码入口：可折叠尺码表（身高/体重矩阵）
- 尺码数量列表：XS–XXL，`- 0 +` 数量控件
- 地址摘要卡片：保存地址后显示（含编辑入口）
- 备注输入框
- 温馨提示条：定制品不退换
- 底部操作栏：分享 / 领券结算 / 加入购物车（UI 交互）
- 生产单导出：保留原有 PDF 生成与预览

## 地址页模块
- 表单字段：姓名、手机号、所在地区、详细地址、设为默认
- 入口按钮：定位 / 智能识别（占位交互）
- 操作按钮：
  - 保存地址（返回结算页）
  - 导入微信地址（仅提示）

## 交互要点
- 点击“物流配送”即进入地址页
- 保存地址后返回结算页并展示摘要卡片
- 尺码表单元可高亮并提示“推荐尺码”
- 数量变更实时更新合计，若全为 0 提示“请选择尺码”
- 领券/加入购物车/分享仅做轻提示，不阻断流程

## 数据与状态
- `view`: 'checkout' | 'address'
- `deliveryMode`: 'delivery' | 'pickup'
- `selectedPreview`: 'front' | 'back'
- `showSizeGuide`: boolean
- `sizeQuantities`: Record<Size, number>
- `address`: { name, phone, region, detail, isDefault }
- `notes`: string
- `coupon`: placeholder 文案

## 风险与约束
- 不引入路由，避免影响当前导航
- 所有新增功能为前端模拟数据
- 保持“生产单导出”功能完整
